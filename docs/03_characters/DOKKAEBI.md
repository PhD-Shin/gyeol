# DOKKAEBI — 도깨비

> **이 문서가 필요한 순간:**
> - 도깨비가 등장하는 장면을 제작할 때
> - 등장 규칙/금지 연출을 확인할 때
> - 도깨비의 역할을 이해해야 할 때

---

## 기능 정의

> **교란자 / 변수 (Disruptor / Variable)**

도깨비는 캐릭터가 아니다.
**힘(Force)**이다.

---

## 역할

- 리듬을 깬다
- 시스템에 개입한다
- 규칙을 따르지 않는 움직임
- 흐름을 흔든다

> 결의 세계에서 도깨비는 **개입(Intervention)**을 담당한다.
> → [[01_world/RULES#4. 개입]]

---

## 태도

- 조롱하는 듯
- 불안정
- 예측 불가능
- 장난인지 공격인지 모호
- **"어긋남 그 자체"**

---

## 존재 방식

- 캐릭터가 아니라 **힘(Force)**
- 의도를 알 수 없음
- 나타났다 사라짐
- 이유 없이 개입함
- 좋은지 나쁜지 판단하지 않음

---

## 시각적 키워드 (15개)

### 형태
1. 뿔 (단일 또는 복수, 비대칭)
2. 거친 질감 (돌, 나무, 이끼)
3. 불균형한 체형
4. 미완성된 형태
5. 경계가 모호한 실루엣

### 존재감
6. 어둠 속에서 빛나는 눈
7. 연기/안개와 함께 등장
8. 프레임 경계에 걸침
9. 반쯤 보이는 상태
10. 실루엣 강조

### 오브제
11. 방망이 또는 막대 (선택적)
12. 빠른 움직임 잔상
13. 주변이 왜곡되는 효과
14. 불꽃/파티클과 함께
15. 그림자가 본체보다 큼

## 동작 키워드 (15개)

### 움직임
1. 점프 (예측 불가 방향)
2. 회전 (축 없는)
3. 갑작스런 정지
4. 리듬과 어긋난 움직임
5. 비틀기

### 등장/퇴장
6. 숨기 (프레임 밖으로)
7. 갑자기 나타남
8. 번쩍 사라짐
9. 연기 속으로 녹음
10. 반대 방향에서 재등장

### 상호작용
11. 흔들기 (공간/대상을)
12. 가리킴 (의미 모호)
13. 웃는 듯한 포즈
14. 인물을 스쳐지나감
15. 배경 일부를 왜곡시킴

## 등장 규칙

| 규칙 | 값 | 설명 |
|------|-----|------|
| 등장 빈도 | 3-5회/영상 | 너무 많으면 익숙해짐 |
| 등장 길이 | 1-3초 | 길게 보여주지 않음 |
| 등장 위치 | 프레임 가장자리/배경 | 중앙에 오래 있지 않음 |
| 카메라 반응 | 등장 시 흔들림/어긋남 | 도깨비가 리듬을 깬다 |
| 음악 반응 | 리듬 브레이크와 동기화 | 비트가 어긋나는 순간 |

---

## Phase별 등장 전략

| Phase | 등장 방식 | 의미 |
|-------|----------|------|
| Phase A | 프레임 가장자리에서 관찰 | 존재 암시 |
| Phase B | 본격적 개입 시작 | 시스템 흔듦 |
| Phase C | 적극적 등장 | 결의 어긋남 극대화 |
| Phase 0 | 기원 설정 | 첫 등장 |

## 도깨비 기능적 변형

> **중요**: 이것은 "종류"가 아니라 **기능적 변형**이다. 설명 금지.

| 변형 | 기능 | 시각적 힌트 | 사용 시기 |
|------|------|-----------|----------|
| 그림자 도깨비 | 암시, 전조 | 그림자만, 본체 없음 | 등장 전조 |
| 연기 도깨비 | 모호한 개입 | 연기와 함께 출몰 | 경계 장면 |
| 불꽃 도깨비 | 강한 개입 | 불꽃 파티클 동반 | 클라이맥스 |
| 울림 도깨비 | 청각적 존재 | 화면에 없음, 소리만 | 긴장 고조 |

### 변형 사용 원칙
- 한 트랙에 1-2 변형만 사용
- 변형 이름을 작품에 명시 금지
- "그림자 → 연기 → 불꽃" 순서로 강도 증가
- Phase 0에서는 "그림자" 변형만

## 다른 존재와의 관계

### 도깨비 ↔ 인물
- 도깨비가 인물의 결을 흔든다
- 직접 접촉하지 않음
- 인물은 도깨비를 인식하지 못하는 듯
- 스쳐지나가며 영향을 줌

### 도깨비 ↔ 해태
- 직접 충돌하지 않음
- 같은 프레임 등장 드묾
- 긴장감으로만 표현
- 도깨비 = 개입 / 해태 = 관찰

### 도깨비 ↔ 탈
- 탈을 쓴 군중 사이를 지나감
- 탈 하나가 반응할 수 있음
- 군중의 리듬을 깨뜨림

## 색상 연계

도깨비 등장 시 색상:

| 상황 | 악센트 색상 | 비율 |
|------|-----------|------|
| 암시 (프레임 가장자리) | 단청 빨강 `#c23a3a` | 3% |
| 등장 (1-2초) | 단청 빨강 | 5-8% |
| 개입 (리듬 브레이크) | 빨강 + 주황 | 10% |
| 퇴장 | 색상 급격히 감소 | 0% |

### 색상 전환 규칙
- 등장 전: 먹색 기조 유지
- 등장 순간: 악센트 스파이크
- 퇴장 후: 원래 색조로 복귀 (0.5초 내)
- **절대 금지**: 무지개빛, 네온, 밝은 색상

## 등장 예고 신호

도깨비가 등장하기 전 암시:

| 신호 | 설명 | 시점 |
|------|------|------|
| 그림자 움직임 | 본체 없이 그림자만 스침 | 등장 2-3초 전 |
| 연기/안개 | 프레임 가장자리에서 피어오름 | 등장 1-2초 전 |
| 카메라 떨림 | 미세한 흔들림 시작 | 등장 1초 전 |
| 음악 어긋남 | 비트가 살짝 밀리거나 당겨짐 | 등장 0.5초 전 |
| 조명 깜빡임 | 등롱광이 흔들림 | 등장 직전 |

→ 예고 신호는 최대 2개까지만 사용. 과하면 예측 가능해짐.

## 등장 실패 & 수정 가이드

### 실패 유형

| 실패 | 증상 | 원인 |
|------|------|------|
| 친근해 보임 | 불안감 없음 | 등장 너무 길거나 중앙 |
| 익숙해짐 | 임팩트 없음 | 등장 빈도 과다 |
| 전통 느낌 | 민속 소개 같음 | 이미지 선정 문제 |
| 인과 명확 | 등장 이유 보임 | 예고 과다 |

### 수정 방법

| 실패 | 수정 |
|------|------|
| 친근해 보임 | 등장 시간 1초로 축소, 가장자리로 이동 |
| 익숙해짐 | 해당 트랙 등장 삭제, 다음 트랙만 |
| 전통 느낌 | 질감 더 거칠게, 조명 더 어둡게 |
| 인과 명확 | 예고 신호 1개로 축소 |
