# VISUAL_PIPELINE — 제작 파이프라인

> **이 문서가 필요한 순간:**
> - 비주얼 제작을 시작할 때
> - AI 도구를 선택할 때
> - 샷 라이브러리를 구축할 때

---

## 핵심 원칙

> **Midjourney는 정체성을 만들고,
> Sora/Veo는 숨을 불어넣는다.
> 편집이 감독이 된다.**

---

## AI 도구 분업

| 도구 | 역할 | 비율 |
|------|------|------|
| **Midjourney** | 세계관 이미지, 정체성 고정 | 80%+ |
| **Sora/Veo** | 클라이맥스 움직임 | 최대 20% |
| **편집** | 리듬, 동기화, 후처리 | 감독 역할 |

---

## 왜 이런 분업인가?

### Midjourney 우선 이유
- **스타일 고정이 가장 강함**
- 연재형 IP = 일관성 최우선
- 재현 가능한 프롬프트 시스템

### Sora/Veo 제한 이유
- 컷마다 카메라 언어/질감/색 달라질 위험
- 일관성 유지 어려움
- 클라이맥스 한두 컷에만 집중

---

## 영상화 규칙 (필수)

| 규칙 | 값 |
|------|-----|
| 컷 길이 | **4-6초 통일** |
| 모션 | 1컷 = 1모션 (줌/팬/고정) |
| 허용 움직임 | 연기, 안개, 불꽃, 파티클 |
| **금지** | 인물/오브제 변형 |

---

## 샷 라이브러리 전략

> **매 영상마다 새로 만들지 않는다.**
> **축적된 이미지를 재조합한다.**

### 권장 50장 구성

| 카테고리 | 수량 | 내용 |
|----------|------|------|
| 도깨비 (교란자) | 8장 | 다양한 앵글/상태 |
| 해태/궁 (질서) | 6장 | 멀리/가까이/부분 |
| 탈/연극 (사회) | 10장 | 단일/군중/다양한 탈 |
| 인물 (익명) | 10장 | 실루엣/뒷모습/부분 |
| 단청-기계 (시스템) | 10장 | 패턴/깨진 패턴/움직임 |
| 섬/바다/지도 (경계) | 6장 | 풍경/지도/경계선 |

### 인물 샷 가이드
- 얼굴 정면 ❌
- 실루엣/역광 ⭕
- 뒷모습 ⭕
- 손/발 부분 ⭕
- 군중/행렬 ⭕

### 장점
- 제작 속도 상승
- 브랜드 일관성 유지
- 재사용 자산 (IP) 축적
- 새 트랙에서 조합만

---

## 실패 규칙

> **"거의 맞는데" → 버린다**

- 애매한 컷은 과감히 폐기
- 완벽하지 않으면 재생성
- 타협하면 일관성 무너짐

---

## 제작 순서

### 1단계: 샷 리스트 작성
- 25-30컷 계획
- 각 컷의 카테고리 지정
- 기존 라이브러리에서 사용 가능한 것 체크

### 2단계: 신규 이미지 생성 (MJ)
- 부족한 컷만 생성
- 스타일 프롬프트 고정
- 대량 생성 후 엄선

### 3단계: 움직임 컷 생성 (Sora/Veo)
- 클라이맥스 1-2컷만
- 나머지는 MJ + 편집 움직임
- 실패 시 MJ로 대체

### 4단계: 편집
- 비트 동기화
- 움직임 추가 (줌/팬)
- 후처리

## Midjourney 프롬프트 템플릿

### 기본 구조
```
[subject], hanji paper texture, ink wash style,
low-key lighting, [color keywords],
film grain, --ar 16:9 --style raw --v 6
```

### 도깨비
```
shadowy dokkaebi figure, asymmetrical horns,
rough stone texture, hanji paper background,
single glowing eye, ink wash style,
mysterious, unsettling, --ar 16:9 --style raw
```

### 해태
```
massive haetae guardian, lion-dragon hybrid,
bronze and stone texture, symmetrical pose,
fog beneath, watching from above,
ink wash style, solemn, --ar 16:9 --style raw
```

### 탈
```
korean hahoe mask, wooden texture,
asymmetrical expression, empty eye holes,
half shadow, hanji paper background,
unsettling presence, --ar 16:9 --style raw
```

## 버전 관리/파일 명명 규칙

### 폴더 구조

```
/gyeol_assets/
├── /library/           # 샷 라이브러리
│   ├── /dokkaebi/
│   ├── /haetae/
│   ├── /tal/
│   ├── /human/
│   ├── /pattern/
│   └── /landscape/
├── /tracks/            # 트랙별 작업
│   └── /[album]_[track]/
│       ├── /source/    # 원본
│       ├── /edit/      # 편집본
│       └── /export/    # 최종
└── /archive/           # 버린 컷
```

### 파일 명명 규칙

```
[존재]_[상태]_[앵글]_[버전].png

예시:
dokkaebi_shadow_profile_v01.png
haetae_statue_high_v02.png
tal_group_front_v01.png
human_silhouette_back_v03.png
```

### 버전 관리
- **v01-v09**: 작업 중
- **v10+**: 승인됨
- **_final**: 최종 사용

## 재생성/폐기 기준

### 즉시 폐기 조건

| 조건 | 이유 |
|------|------|
| 얼굴 정면 노출 | 인물 규칙 위반 |
| 밝은 조명 | 톤 위반 |
| 네온/파스텔 색상 | 색상 규칙 위반 |
| 선명한 디지털 느낌 | 질감 규칙 위반 |
| 존재 캐릭터화 | 세계관 위반 |

### 재생성 고려 조건

| 조건 | 조치 |
|------|------|
| 구도 약간 틀림 | 크롭으로 수정 시도 |
| 색상 약간 다름 | 후처리로 수정 시도 |
| 질감 부족 | 텍스처 오버레이 추가 |

### "거의 맞는데" 판정 기준
- 1회 수정으로 해결 가능 → 수정
- 2회 이상 필요 → 폐기
- 의심되면 → 폐기

## Phase별 샷 라이브러리 구성

### Phase A 라이브러리 (기본)

| 카테고리 | 수량 | 특징 |
|----------|------|------|
| 도깨비 | 8장 | 암시적, 그림자 |
| 해태 | 4장 | 없음 또는 발견 전 |
| 탈 | 10장 | 소규모 군중 (1-5) |
| 인물 | 10장 | 실루엣 위주 |
| 패턴 | 8장 | 완벽한 반복 |
| 풍경 | 10장 | 황혼, 안개 |

### Phase B 라이브러리 (확장)

| 카테고리 | 수량 | 특징 |
|----------|------|------|
| 도깨비 | 12장 | 활발, 다양한 변형 |
| 해태 | 8장 | 등장, 관찰 |
| 탈 | 15장 | 대규모 군중 (5-20) |
| 인물 | 8장 | 부분 노출 |
| 패턴 | 10장 | 깨진 패턴 포함 |
| 풍경 | 8장 | 경계 모호 |

### Phase C 라이브러리 (흔적)

| 카테고리 | 수량 | 특징 |
|----------|------|------|
| 도깨비 | 6장 | 감소, 흔적만 |
| 해태 | 10장 | 판단, 기동 |
| 탈 | 8장 | 인물과 섞임 |
| 인물 | 6장 | 경계 모호 |
| 패턴 | 6장 | 흐려진 패턴 |
| 풍경 | 10장 | 기록, 유적 |

### Phase 0 라이브러리 (기원)

| 카테고리 | 수량 | 특징 |
|----------|------|------|
| 존재들 | 8장 | 각각 분리, 첫 등장 |
| 인물 | 4장 | 형체만 |
| 패턴 | 4장 | 원시적 |
| 풍경 | 8장 | 태초, 무시간 |

## 에러 복구 프로토콜

### AI 생성 실패 시

| 실패 유형 | 1차 조치 | 2차 조치 |
|----------|----------|----------|
| 스타일 불일치 | 프롬프트 조정 | 시드 고정 + 재생성 |
| 구도 문제 | 크롭/리프레임 | 재생성 |
| 품질 문제 | Upscale | 재생성 |
| 존재 왜곡 | 즉시 폐기 | 라이브러리 대체 |

### 데드라인 긴급 시
1. 샷 라이브러리에서 유사 컷 검색
2. 기존 컷 색상/크롭 변형
3. 새 컷 수 줄이기 (필수 컷만)

## 체크리스트

파이프라인 완성 전:

- [ ] Sora/Veo가 20% 이하인가?
- [ ] 컷 길이 4-6초인가?
- [ ] 인물/오브제 변형이 없는가?
- [ ] 샷 라이브러리를 활용했는가?
