# QC_CHECKLIST — 품질 검사표

> **이 문서가 필요한 순간:**
> - 트랙 완성 후 품질 검사할 때
> - 공개 전 최종 확인할 때
> - 문제점 발견 시 체크할 때

---

## 사용법

1. 트랙 완성 후 이 체크리스트 순서대로 확인
2. 모든 항목 통과해야 공개 가능
3. 실패 항목은 수정 후 재검사
4. 최종 검사자 서명 후 완료

---

## 1. 세계관 일관성 검사

### 1.1 금지 규칙

| 항목 | 통과 | 메모 |
|------|------|------|
| 역사/정치 직접 언급 없음 | ☐ | |
| 지명/국가명 없음 | ☐ | |
| 특정 사건 직구 없음 | ☐ | |
| 민속 설명 없음 | ☐ | |
| 감정 직접 설명 없음 | ☐ | |
| 희망/구원 메시지 없음 | ☐ | |
| 일본 도상 없음 | ☐ | |

### 1.2 테마 연결

|------|------|------|
| 메타 테마 1-2개 연결됨 | ☐ | |
| Phase에 맞는 메시지 강도 | ☐ | |
| 작동 규칙 중 1개 이상 표현 | ☐ | |

### 1.3 존재 규칙

| 항목 | 통과 | 메모 |
|------|------|------|
| 도깨비: 기능만 표현 | ☐ | |

## 6. 최종 확인

### 6.1 전체 시청 테스트

- [ ] 처음부터 끝까지 시청 완료
- [ ] 몰입 깨지는 순간 없음
- [ ] 결의 톤 유지됨
- [ ] "쿨하다" 느낌 유지됨

### 6.2 외부 피드백

- [ ] 1명 이상 피드백 수집
- [ ] 피드백 반영 또는 기각 결정
