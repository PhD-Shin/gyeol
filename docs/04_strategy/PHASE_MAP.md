# PHASE_MAP — Phase 로드맵

> **이 문서가 필요한 순간:**
> - 트랙이 어느 Phase에 속하는지 결정할 때
> - 메시지 강도를 확인할 때
> - 전체 프로젝트 로드맵을 파악할 때

---

## 핵심 원칙

> **Phase A에서 직구는 금물이다.**
> **상징으로 시작하고, 해석은 팬에게 위임한다.**

## Phase 개요

| Phase | 이름 | 핵심 | 메시지 강도 | 앨범 | 트랙 |
|-------|------|------|-----------|------|------|
| Phase A | 개입 (Intervention) | 시스템과 어긋남 | 상징적, 보편적 | 2 | 16 |
| Phase B | 경계 (Border) | 지워지는 선 | 은유적 | 2 | 16 |
| Phase C | 기록 (Record) | 남는 것과 지워지는 것 | 점진적 명시 | 2 | 16 |

> **Phase 0 (기원)**: 반응 좋으면 보너스로 추가 (선택)

## Phase A — 개입 (Intervention)

### 핵심 질문
> **"시스템이 완벽할 때, 어긋남은 어디서 오는가?"**

### 허용 메시지 강도
- 상징적, 보편적
- 특정 역사/정치 언급 ❌
- 글로벌 누구나 공감 가능

### 시각적 특징
- 도깨비 등장 시작
- 완벽한 반복 → 미세한 어긋남
- 어둠 위주, 악센트 최소

### 테마 강조
- System vs Deviation
- Ritual
- Repetition

### 트랙 예시
- 흔들림 (HEUNDEULM)
- 틈 (TIL)
- 균열 (GYUNYEOL)

---

## Phase B — 경계 (Border)

### 핵심 질문
> **"선이 지워질 때, 안과 밖은 어떻게 되는가?"**

### 허용 메시지 강도
- 은유적
- "경계", "지움" 개념 등장
- 여전히 직접 언급 ❌

### 시각적 특징
- 먹 번짐 증가
- 경계 모호한 공간
- 빨강 악센트 증가

### 테마 강조
- Border
- Erasure
- Name

### 트랙 예시
- 경계 (GYEONGGYE)
- 지움 (JIUM)
- 이름 (IREUM)

---

## Phase C — 기록 (Record)

### 핵심 질문
> **"남는 것은 무엇이고, 지워지는 것은 무엇인가?"**

### 허용 메시지 강도
- 점진적 명시
- 커뮤니티 포스트로 "영감" 언급 가능
- 여전히 직구 ❌, 상징 중심

### 시각적 특징
- 지도, 도장, 흔적
- 해태 등장 증가
- 의례/완료 장면

### 테마 강조
- Record
- Name
- Disappearance

### 트랙 예시
- 기록 (GIROK)
- 도장 (DOJANG)
- 흔적 (HEUNJEOK)

---

## Phase 0 — 기원 (Origin) [선택적]

> **반응 좋으면 보너스로 추가**

### 핵심 질문
> **"모든 것이 시작되기 전, 무엇이 있었는가?"**

### 허용 메시지 강도
- 가장 추상적
- 시간 이전
- 선언적, 신화적

### 시각적 특징
- 극단적 어둠
- 단일 광원
- 존재들의 첫 등장

### 테마 강조
- Time
- Movement
- 모든 테마의 기원

### 제작 조건
- Phase C 완료 후 반응 평가
- 구독자 50,000+ 시 제작 검토
- 1 앨범 × 6-8 트랙

---

## Phase 전환 조건

| 전환 | 조건 | 타이밍 |
|------|------|--------|
| A → B | 코어 팬덤 형성, 구독자 안정 | A-2 완료 후 (16주차) |
| B → C | 팬덤이 해석 시작, 커뮤니티 활성 | B-2 완료 후 (32주차) |
| C → 0 | IP 확립, 세계관 확장 준비 | C-2 완료 후 (48주차), 선택적 |

### 반응별 대응

| 반응 | 대응 |
|------|------|
| 좋음 | 계획대로 진행 |
| 보통 | 축소 (앨범당 6트랙) |
| 없음 | Phase A 완료 후 중단 검토 |

## Phase 전환 브릿지

### 음악적 브릿지

| 전환 | 마지막 트랙 특징 | 첫 트랙 연결 |
|------|----------------|-------------|
| A → B | 리듬 불안정, 페이드 아웃 없이 끊김 | 경계 모호한 인트로 |
| B → C | 저음 드론으로 마무리 | 기록/흔적 사운드로 시작 |
| C → 0 | 모든 소리 사라짐 | 태초의 단일 음 |

### 시각적 브릿지

| 전환 | 마지막 MV 엔딩 | 첫 MV 오프닝 |
|------|---------------|-------------|
| A → B | 도깨비 시선 → 안개로 전환 | 안개 속에서 시작 |
| B → C | 지워지는 경계선 | 남은 흔적에서 시작 |
| C → 0 | 모든 것 사라짐 후 여백 | 어둠 속 단일 광원 |

### 공유 테마 (전환점)

| 전환 | 공유 테마 | 브릿지 역할 |
|------|----------|-----------|
| A-B | Deviation | A의 어긋남 → B의 경계 붕괴 |
| B-C | Erasure | B의 지움 → C의 기록 (역설) |
| C-0 | Time | C의 흔적 → 0의 시작 전 |

## Season 종료 콘텐츠

### Season 클라이맥스 트랙

각 Season 마지막 트랙은 **특별 포맷**:
- 길이: 4-5분 (일반 3분보다 김)
- 모든 존재 최소 1회 등장
- 비주얼라이저 확장판 (30분) 별도 공개

### Season 완결 콘텐츠

| Season | 완결 특별 콘텐츠 |
|--------|---------------|
| Season 1 (Phase A) | "Making of" 쇼츠 시리즈 (5-7개) |
| Season 2 (Phase B) | 커뮤니티 포스트 "숨겨진 연결" |
| Season 3 (Phase C) | 전체 타임라인 영상 (팬 해석 종합) |

### 다음 Season 티저

Season 종료 1주일 후:
- 다음 Season 30초 티저 공개
- 새 존재/테마 암시
- 카운트다운 시작

## Phase별 금지 사항

### Phase A (엄격)
- 지명/국가명 ❌
- 역사 직구 ❌
- 정치 논쟁 ❌

### Phase B (완화)
- 직접 언급은 여전히 ❌
- "경계", "이름" 개념은 ⭕

### Phase C (점진적)
- 비하인드로 "영감" 언급 가능
- 본 콘텐츠는 여전히 상징 중심

## 운영 전략

### Phase A 운영
- 글로벌 타겟
- 댓글 모니터링 필수
- 논쟁 유발 시 즉시 대응

### Phase B-C 운영
- 커뮤니티 포스트로 점진적 공개
- 팬 해석 장려
- 공식 입장은 중립 유지

## Phase 전환 실패 시나리오

### 전환 실패 유형

| 실패 | 증상 | 원인 | 대응 |
|------|------|------|------|
| 조기 전환 | 팬덤 혼란, 맥락 없음 | 구독자 목표 도달 전 전환 | 현 Phase 연장 |

| 팬덤 이탈 | 구독자 감소 | Phase 방향 맞지 않음 | 팬 피드백 분석 |

### Phase 롤백 규칙

| 상황 | 롤백 가능 | 방법 |
|------|----------|------|
| 트랙 1-2개 공개 후 | ⭕ | 트랙 삭제 + 이전 Phase 연장 |
| 앨범 50% 이상 공개 | ❌ | 현 Phase 내 방향 수정 |
| Season 전환 후 | ❌ | 다음 Phase에서 보정 |

## Phase 전환 결정 프로세스

### 전환 결정 체크리스트

```
□ KPI 달성 여부 확인
□ 현 Phase 앨범 완성 여부
□ 다음 Phase 자산 준비 상태
□ 팬덤 반응 분석 완료
□ 브릿지 콘텐츠 준비 완료
```

### 전환 타임라인

| 단계 | 시점 | 액션 |
|------|------|------|
| 검토 | 앨범 종료 2주 전 | KPI 검토 |
| 결정 | 앨범 종료 1주 전 | 전환 여부 결정 |
| 준비 | 앨범 종료 | 브릿지 콘텐츠 준비 |
| 전환 | 앨범 종료 2주 후 | 다음 Phase 시작 |

## 크로스 Phase 규칙

### 허용되는 크로스 Phase 요소

| 요소 | 규칙 | 예시 |
|------|------|------|
| 시각적 암시 | 이전 Phase 모티프 배경 등장 | A의 그림자가 B에서 잠깐 스침 |
| 음악적 인용 | 2-4마디만 변주 | A 멜로디가 C에서 역재생 |
| 색상 참조 | Phase 전환점에서만 | A 악센트가 B 첫 트랙에 잔존 |

### 금지되는 크로스 Phase 요소

| 금지 | 이유 |
|------|------|
| 직접 스토리 연결 ❌ | Phase 독립성 침해 |
| 존재의 "성장/변화" 암시 ❌ | 캐릭터화 금지 |
| 이전 Phase 설명 ❌ | 결은 설명하지 않음 |

## 체크리스트

Phase 결정 전:

- [ ] 이 트랙의 Phase가 명확한가?
- [ ] 메시지 강도가 Phase에 맞는가?
- [ ] 금지 사항을 위반하지 않는가?
- [ ] 시각적 특징이 Phase와 맞는가?
- [ ] 테마 강조가 적절한가?

Phase 전환 전:

- [ ] KPI 달성 여부 확인했는가?
- [ ] 브릿지 콘텐츠가 준비되었는가?
- [ ] 다음 Phase 자산이 준비되었는가?
- [ ] 롤백 필요성을 검토했는가?
